# SPRING BOOT — INTERVIEW REVISION GUIDE

## PAGE 1: INTRODUCTION & ARCHITECTURE

Spring Boot is an open-source Java framework built on top of the Spring Framework that helps to create **stand-alone, production-ready applications** quickly with minimal configuration.

---

# QUICK RECAP

| Property | Value |
|---|---|
| What | Rapid application development framework |
| Built On | Spring Framework |
| Language | Java |
| Purpose | Build stand-alone, production-grade Spring applications easily |
| Key Benefit | Convention over configuration |
| Default | Embedded server, auto configuration, starter dependencies |

---

# 1. WHAT IS SPRING BOOT?

- Spring Boot simplifies Spring application development.
- It provides default configurations, embedded servers, and production-ready features.
- It follows the **"Convention over Configuration"** approach.
- It allows you to focus on business logic instead of infrastructure.
- It can create stand-alone applications that can run independently.

---

# 2. WHY SPRING BOOT?

- Reduces development time
- Minimizes configuration
- No need for XML configuration
- Embedded servers (Tomcat, Jetty, Undertow)
- Easy dependency management
- Production-ready features (Actuator, Health checks, Metrics)
- Easier testing
- Great for Microservices & Cloud applications

---

# 3. PROBLEMS SOLVED BY SPRING BOOT

- Complex XML-based configurations
- Tedious dependency management
- Deployment as WAR file in external server
- Version conflicts
- Lack of production-ready features
- Difficult environment-specific configuration

---

# 4. SPRING VS SPRING BOOT

| Spring Framework | Spring Boot |
|---|---|
| Provides core framework and many modules | Built on top of Spring Framework |
| Needs extensive configuration | Auto configuration by default |
| No embedded server by default | Embedded server (Tomcat/Jetty/Undertow) |
| Deployed as WAR | Deployed as executable JAR (stand-alone) |
| Requires manual dependency management | Starter dependencies simplify management |
| More setup, more XML | Less setup, minimal configuration |

---

# 5. KEY FEATURES OF SPRING BOOT

## Auto Configuration
Automatically configures Spring application based on dependencies and classpath.

## Starter Dependencies
Provide pre-configured dependencies for specific functionalities.

Examples:
- `spring-boot-starter-web`
- `spring-boot-starter-data-jpa`
- `spring-boot-starter-security`

## Embedded Servers
Comes with embedded:
- Tomcat
- Jetty
- Undertow

No need to deploy WAR externally.

## Opinionated
Follows best practices and conventions to reduce developer choices.

## Production Ready
Actuator, health checks, metrics, and monitoring endpoints out of the box.

## Stand-alone
Applications are packaged as executable JARs and can run independently.

# 6. SPRING BOOT ARCHITECTURE

## Flow Overview

1. Client sends a request to the application.
2. Embedded server (Tomcat/Jetty/Undertow) receives the request.
3. DispatcherServlet handles the request and routes it to the appropriate controller.
4. Spring Context provides the required beans (services, repositories, etc.).
5. Response is generated and sent back to the client.

---

## Architecture Components

### Client
Sends REST API requests.

### Embedded Server
Handles incoming HTTP requests.

Examples:
- Tomcat
- Jetty
- Undertow

### DispatcherServlet
Acts as Front Controller.

### Controller Layer (`@Controller`)
Handles requests.

### Service Layer (`@Service`)
Contains business logic.

### Repository Layer (`@Repository`)
Handles database operations.

### Spring Context / IoC Container
- Creates and manages beans
- Handles dependency injection
- Supports AOP and transactions

### Data Source
Examples:
- MySQL
- PostgreSQL
- H2

---

## Spring Boot Startup Flow

1. `SpringApplication.run()` starts the application.
2. Auto Configuration configures beans based on classpath and properties.
3. Component Scanning finds components:
   - `@Component`
   - `@Service`
   - `@Controller`
   - `@Repository`
4. IoC Container creates beans and manages lifecycle.
5. Application becomes ready to handle requests.

# 7. AUTO CONFIGURATION

Spring Boot auto-configures your application based on:

- Dependencies in classpath
- Configuration in `application.properties` / `application.yml`
- Default conventions

## Example

If `spring-boot-starter-web` is present in classpath, Spring Boot automatically configures:

- Embedded Tomcat
- DispatcherServlet
- Jackson (for JSON)
- Default error handling
- Much more...

---

# 8. STARTER DEPENDENCIES

Starters are a set of convenient dependency descriptors.

## Popular Starters

| Starter | Purpose |
|---|---|
| `spring-boot-starter-web` | Web applications |
| `spring-boot-starter-data-jpa` | JPA + Hibernate |
| `spring-boot-starter-security` | Spring Security |
| `spring-boot-starter-test` | Testing support |
| `spring-boot-starter-actuator` | Monitoring |
| `spring-boot-starter-validation` | Validation |

### Benefit
Add one starter and get all required dependencies automatically.

---

# 9. EMBEDDED SERVERS

Spring Boot comes with embedded servers.

No need to deploy WAR file.

| Server | Artifact | Default Port | Notes |
|---|---|---|---|
| Tomcat | `spring-boot-starter-tomcat` | 8080 | Most widely used |
| Jetty | `spring-boot-starter-jetty` | 8080 | Lightweight |
| Undertow | `spring-boot-starter-undertow` | 8080 | High performance |

## Change Default Port

```properties
server.port=9090
```

---

# 10. OPINIONATED CONFIGURATION

Spring Boot provides sensible defaults.

You can override whenever needed.

## Example Defaults

- Default port: `8080`
- Default static locations
- Default error handling
- Default JSON mapper
- Default logging
- Default security (basic auth for actuator)

# KEY TAKEAWAYS

- Spring Boot simplifies Spring development with convention over configuration.
- It provides out-of-the-box features for rapid application development.
- Starters, auto configuration, and embedded servers are the core elements.
- It is production-ready and widely used for Microservices and Cloud Native applications.
- Spring Boot helps build robust, scalable, and maintainable applications quickly.
