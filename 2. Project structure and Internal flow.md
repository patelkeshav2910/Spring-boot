# 2. @SPRINGBOOTAPPLICATION ANNOTATION

```java
@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

It is a combination of 3 annotations:

| Annotation | Purpose |
|---|---|
| `@Configuration` | Marks the class as source of bean definitions |
| `@EnableAutoConfiguration` | Enables Spring Boot auto configuration |
| `@ComponentScan` | Scans current package and sub-packages |

---

# 3. MAIN() METHOD & APPLICATION STARTUP FLOW

## Startup Steps

1. JVM starts the application.
2. `main()` method is called.
3. `SpringApplication.run()` is invoked.
4. SpringApplication initializes the environment.
5. Bean definitions are loaded from classpath & configuration.
6. `ApplicationContext` is created (IoC Container).
7. Beans are instantiated, configured, and initialized.
8. Embedded server starts and application becomes ready.

## Key Point

Spring Boot creates and manages the entire application context automatically.

# 4. COMPONENT SCANNING

Spring Boot automatically scans:

- Package of `@SpringBootApplication`
- All sub-packages

## It scans for:

- `@Component`
- `@Service`
- `@Repository`
- `@Controller`
- `@RestController`
- `@Configuration`

## Example Package Structure

```text
com.example.demo
│
├── controller
├── service
├── repository
└── model
```

All components inside these packages are registered as Spring Beans.

## Best Practice

Keep the main class in the root package.

# 5. BEAN CREATION & LIFECYCLE

## Bean Lifecycle Flow

1. Bean Definition Loaded
2. Bean Instantiation
3. Dependency Injection
4. Bean Initialization
5. Bean Ready to Use
6. Bean Destruction

---

## Important Lifecycle Hooks

| Annotation / Interface | Purpose |
|---|---|
| `@PostConstruct` | Runs after bean initialization |
| `InitializingBean` | Custom initialization logic |
| `@PreDestroy` | Runs before bean destruction |
| `DisposableBean` | Cleanup logic |

Lifecycle is managed by Spring IoC Container.

# 6. IOC CONTAINER

Spring Boot uses IoC (Inversion of Control) Container to manage:

- Beans
- Dependencies
- Lifecycle

## IoC Benefits

- Loose coupling
- Easier testing
- Better maintainability
- Centralized component management

---

## IoC Flow

```text
Bean Object
    ↓
IoC Container (Spring Context)
    ↓
Dependency Injection
```

The container provides and injects required dependencies automatically.

---

# 7. DEPENDENCY INJECTION

IoC Container injects required dependencies into beans.

## Types of Dependency Injection

1. Constructor Injection ✅ Recommended
2. Setter Injection
3. Field Injection ❌ Not Recommended

---

## Constructor Injection Example

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

## Why Constructor Injection?

- Ensures immutability
- Prevents `NullPointerException`
- Makes dependencies mandatory
- Easier to test

# 8. MAVEN VS GRADLE

| Maven (`pom.xml`) | Gradle (`build.gradle`) |
|---|---|
| XML based | Groovy / Kotlin DSL |
| Convention over configuration | Flexible & powerful |
| Build lifecycle predefined | Incremental build (faster) |
| Better for beginners | Better for large projects |
| Large community & adoption | Less XML, more readable |

## Common Point

Both are build tools used for:

- Dependency management
- Build
- Testing
- Packaging

# 9. FAT JAR (UBER JAR)

Spring Boot packages:

- Project classes
- Libraries
- Dependencies
- Resources

into a single executable JAR.

---

## Run Application

```bash
java -jar application.jar
```

## Benefits

- Easy deployment
- No need to install external servers
- Self-contained configuration
- Same behavior everywhere

---

# 10. WHAT HAPPENS INTERNALLY AFTER SpringApplication.run()

## Internal Flow

### 1. Environment Prepared
- Loads properties from:
  - `application.properties`
  - `application.yml`
- Sets active profiles

### 2. ApplicationContext Created
- Creates IoC Container
- Prepares bean factory

### 3. Component Scanning
- Scans packages
- Registers components as beans

### 4. Auto Configuration Applied
- Configures beans automatically
- Based on:
  - Classpath
  - Properties
  - Conditions

### 5. Beans Instantiated & Wired
- Dependencies injected
- Beans initialized
- Post-processors applied

### 6. Embedded Server Starts
- Tomcat / Jetty / Undertow starts
- Application becomes ready to accept requests

# KEY TAKEAWAYS

- Spring Boot reduces configuration and boilerplate code.
- It provides embedded servers and auto configuration.
- IoC Container manages beans and dependencies.
- Component scanning automatically detects components.
- Fat JAR makes deployment simple and portable.
