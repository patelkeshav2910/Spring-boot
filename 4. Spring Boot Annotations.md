# SPRING BOOT — INTERVIEW REVISION GUIDE

## PAGE 4: SPRING BOOT ANNOTATIONS DEEP DIVE

---

# QUICK RECAP

- Annotations are the foundation of Spring Boot.
- They are used for:
  - Configuration
  - Bean definition
  - Request handling
  - Dependency injection
  - Transactions
  - Validation
- Knowing when and where to use each annotation is important in interviews.

---

# 1. ANNOTATION CATEGORIES OVERVIEW

| Category | Purpose | Examples |
|---|---|---|
| Core / Boot Annotations | Bootstrap and configure Spring Boot | `@SpringBootApplication` |
| Component Annotations | Declare Spring-managed beans | `@Component`, `@Service` |
| Web Annotations | Handle REST APIs and web requests | `@RestController`, `@GetMapping` |
| Bean / Configuration Annotations | Define beans and configuration | `@Configuration`, `@Bean` |
| Transaction Annotations | Transaction management | `@Transactional` |
| Validation Annotations | Validate request/input data | `@Valid`, `@NotNull` |

---

# 2. CORE & BOOT ANNOTATIONS

| Annotation | Purpose | Used On | Example / Notes |
|---|---|---|---|
| `@SpringBootApplication` | Enables auto-configuration, component scanning, and configuration properties | Main Class | Main entry annotation |
| `@SpringBootConfiguration` | Indicates configuration class | Class | Used internally by `@SpringBootApplication` |
| `@EnableAutoConfiguration` | Enables Spring Boot auto configuration | Class | Automatically configures beans |
| `@ComponentScan` | Scans packages for Spring components | Class | `@ComponentScan("com.example")` |
| `@ConfigurationProperties` | Binds external properties to Java object | Class | `@ConfigurationProperties(prefix="app")` |
| `@EnableConfigurationProperties` | Enables `@ConfigurationProperties` classes | Class | Registers config property beans |

---

## Example

```java
@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

---

# 3. COMPONENT ANNOTATIONS

| Annotation | Purpose | Used On | Notes |
|---|---|---|---|
| `@Component` | Generic Spring-managed bean | Class | Base annotation |
| `@Service` | Service layer bean | Class | Business logic layer |
| `@Repository` | DAO / Repository layer | Class | Handles DB exceptions |
| `@Controller` | MVC Controller | Class | Returns HTML views |
| `@RestController` | REST Controller (`@Controller + @ResponseBody`) | Class | Returns JSON/XML |
| `@ComponentScan` | Scans components in packages | Class | Used explicitly if needed |

---

## Tip

Prefer specific stereotypes:
- `@Service`
- `@Repository`
- `@Controller`

instead of generic `@Component`.

---

# 4. WEB ANNOTATIONS

| Annotation | Purpose | Used On | Example |
|---|---|---|---|
| `@RequestMapping` | Maps HTTP requests | Class / Method | `@RequestMapping("/api")` |
| `@GetMapping` | Handles GET requests | Method | `@GetMapping("/users/{id}")` |
| `@PostMapping` | Handles POST requests | Method | `@PostMapping("/users")` |
| `@PutMapping` | Handles PUT requests | Method | `@PutMapping("/users/{id}")` |
| `@DeleteMapping` | Handles DELETE requests | Method | `@DeleteMapping("/users/{id}")` |
| `@PathVariable` | Reads path variable | Parameter | `@PathVariable Long id` |
| `@RequestParam` | Reads query parameter | Parameter | `@RequestParam String name` |
| `@RequestBody` | Binds request body | Parameter | `@RequestBody User user` |
| `@ResponseBody` | Returns data directly as response | Method | JSON/XML response |
| `@ResponseEntity` | Represents complete HTTP response | Method | `ResponseEntity.ok(data)` |

---

# 5. BEAN / CONFIGURATION ANNOTATIONS

| Annotation | Purpose | Used On | Example |
|---|---|---|---|
| `@Configuration` | Defines configuration class | Class | Bean definitions |
| `@Bean` | Declares Spring Bean | Method | `@Bean public MyService service()` |
| `@Value` | Injects value from properties | Field / Parameter | `@Value("${app.name}")` |
| `@PropertySource` | Loads property file | Class | `@PropertySource("classpath:app.properties")` |
| `@Import` | Imports configuration classes | Class | `@Import(AppConfig.class)` |
| `@Profile` | Loads beans for specific profile | Class / Method | `@Profile("dev")` |

---

# 6. TRANSACTION ANNOTATIONS

| Annotation | Purpose | Used On | Notes |
|---|---|---|---|
| `@Transactional` | Defines transaction boundary | Class / Method | Rollback on `RuntimeException` |
| `@EnableTransactionManagement` | Enables annotation-driven transactions | Configuration Class | Usually auto-enabled |
| `@Transactional(readOnly = true)` | Optimized read-only transaction | Method | Improves performance |

---

# 7. VALIDATION ANNOTATIONS (JAKARTA VALIDATION)

| Annotation | Purpose | Example |
|---|---|---|
| `@Valid` | Triggers validation | `@Valid @RequestBody User user` |
| `@NotNull` | Field must not be null | `@NotNull private String name;` |
| `@NotBlank` | String must not be blank | `@NotBlank private String email;` |
| `@Size(min,max)` | Size range validation | `@Size(min = 3, max = 20)` |
| `@Email` | Valid email format | `@Email private String email;` |
| `@Pattern` | Regex validation | `@Pattern(regexp = "[A-Z]{3}")` |

# 8. ANNOTATION HIERARCHY & RELATIONSHIP

---

## Core

```text
@SpringBootApplication
    ├── @SpringBootConfiguration
    ├── @EnableAutoConfiguration
    └── @ComponentScan
```

---

## Component Stereotypes

```text
@Component
    ├── @Service
    ├── @Repository
    └── @Controller
            └── @RestController
```

---

## Web Mapping Hierarchy

```text
@RequestMapping
    ├── @GetMapping
    ├── @PostMapping
    ├── @PutMapping
    └── @DeleteMapping
```

---

## Configuration Annotations

```text
@Configuration
    ├── @Bean
    ├── @Value
    ├── @PropertySource
    └── @Profile
```

# 9. WHEN TO USE WHICH ANNOTATION?

| Scenario | Annotation |
|---|---|
| Creating main application class | `@SpringBootApplication` |
| Creating service class | `@Service` |
| Creating repository class | `@Repository` |
| Creating REST API | `@RestController` + Mapping annotations |
| Injecting configuration values | `@Value` or `@ConfigurationProperties` |
| Managing transactions | `@Transactional` |
| Validating request body | `@Valid` + validation annotations |

# 11. KEY TAKEAWAYS

- Annotations reduce XML configuration and boilerplate code.
- Choose the correct annotation for the correct layer.
- Spring scans and creates beans using annotations.
- Many annotations are built on top of others.
- Annotation knowledge is heavily asked in interviews.

---

  - When to use it

- Combine annotations smartly for:
