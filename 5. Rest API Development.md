# 1. REST ARCHITECTURAL PRINCIPLES

| Principle | Description |
|---|---|
| Client-Server | Separation of concerns between client and server |
| Stateless | Each request contains all required information. No session stored |
| Cacheable | Responses should define whether they are cacheable |
| Uniform Interface | Consistent interface simplifies architecture |
| Layered System | Client does not know whether connected directly to server or intermediary |
| Code on Demand (Optional) | Server can extend client functionality temporarily |

---

# 2. HTTP METHODS IN REST

| Method | Purpose | Idempotent | Safe |
|---|---|---|---|
| GET | Retrieve resource | ✅ Yes | ✅ Yes |
| POST | Create resource | ❌ No | ❌ No |
| PUT | Update/Replace resource | ✅ Yes | ❌ No |
| PATCH | Partial update | ❌ No | ❌ No |
| DELETE | Delete resource | ✅ Yes | ❌ No |

---

## Important Concepts

### Idempotent
Multiple identical requests produce same result as one request.

### Safe
Request does not modify server state.

---

# 3. REQUEST FLOW IN SPRING BOOT REST API

```text
Client
   ↓
HTTP Request
   ↓
DispatcherServlet
   ↓
Controller Handler Method
   ↓
Response (JSON/XML + Status Code)
```

---

## Key Points

- `DispatcherServlet` acts as Front Controller.
- Routes request to appropriate controller method.
- Controller processes request and returns response.
- Response converted to JSON/XML using `HttpMessageConverter`.

# 4. MAPPING ANNOTATIONS

| Annotation | HTTP Method | URL Example | Description |
|---|---|---|---|
| `@GetMapping` | GET | `/api/users` | Retrieve resources |
| `@PostMapping` | POST | `/api/users` | Create resource |
| `@PutMapping` | PUT | `/api/users/1` | Update/replace resource |
| `@PatchMapping` | PATCH | `/api/users/1` | Partial update |
| `@DeleteMapping` | DELETE | `/api/users/1` | Delete resource |

---

## Example Controller Methods

### GET

```java
@GetMapping("/users")
public List<User> getAllUsers() {
    return service.getAllUsers();
}
```

---

### POST

```java
@PostMapping("/users")
public User createUser(@RequestBody User user) {
    return service.create(user);
}
```

---

### PUT

```java
@PutMapping("/users/{id}")
public User updateUser(
        @PathVariable Long id,
        @RequestBody User user) {

    return service.update(id, user);
}
```

---

### DELETE

```java
@DeleteMapping("/users/{id}")
public void deleteUser(@PathVariable Long id) {
    service.delete(id);
}
```

---

# BEST PRACTICES FOR URL DESIGN

✅ Use nouns, not verbs

```text
/users
```

❌ Avoid:

```text
/getUsers
```

---

✅ Use plural naming

```text
/users
```

❌ Avoid:

```text
/user
```

---

✅ Use proper HTTP methods

- GET → Read
- POST → Create
- PUT/PATCH → Update
- DELETE → Delete

---

✅ Version APIs

```text
/api/v1/users
```

# 5. PATH VARIABLE VS REQUEST PARAM

| Feature | `@PathVariable` | `@RequestParam` |
|---|---|---|
| Used For | Values from URL path | Query parameters |
| Example URL | `/api/users/101` | `/api/users?page=1&size=10` |

---

## PathVariable Example

```java
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) {
    return service.getById(id);
}
```

---

## RequestParam Example

```java
@GetMapping("/users")
public List<User> getUsers(
        @RequestParam int page,
        @RequestParam int size) {

    return service.getUsers(page, size);
}
```

---

## Important Difference

- `@PathVariable` → Part of path
- `@RequestParam` → Part of query string

# 6. @REQUESTBODY & @RESPONSEBODY

---

## @RequestBody

Binds HTTP request body to Java object.

Used with:
- POST
- PUT
- PATCH

---

### Example JSON Request

```json
{
  "name": "Alice",
  "email": "alice@gmail.com"
}
```

---

### Controller Example

```java
@PostMapping("/users")
public User createUser(@RequestBody User user) {
    return service.create(user);
}
```

---

## @ResponseBody

Converts Java object into HTTP response body (JSON/XML).

Used implicitly with `@RestController`.

---

### Example JSON Response

```json
{
  "id": 101,
  "name": "Alice",
  "email": "alice@gmail.com"
}
```

---

## Important Note

```text
@RestController = @Controller + @ResponseBody
```

# 7. RESPONSEENTITY & STATUS CODES

`ResponseEntity` gives full control over:

- Status code
- Headers
- Response body

---

## Example

```java
@GetMapping("/users/{id}")
public ResponseEntity<User> getUser(
        @PathVariable Long id) {

    User user = service.getUser(id);

    if (user == null) {
        return ResponseEntity.notFound().build();
    }

    return ResponseEntity.ok(user);
}
```

# 8. SAMPLE REST CONTROLLER (CRUD)

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @Autowired
    private UserService service;

    // GET All Users
    @GetMapping
    public List<User> getAllUsers() {
        return service.getAllUsers();
    }

    // GET User By Id
    @GetMapping("/{id}")
    public ResponseEntity<User> getUserById(
            @PathVariable Long id) {

        return ResponseEntity.ok(
                service.getUser(id));
    }

    // CREATE User
    @PostMapping
    public ResponseEntity<User> createUser(
            @RequestBody User user) {

        User saved = service.create(user);

        return ResponseEntity
                .status(HttpStatus.CREATED)
                .body(saved);
    }

    // UPDATE User
    @PutMapping("/{id}")
    public ResponseEntity<User> updateUser(
            @PathVariable Long id,
            @RequestBody User user) {

        return ResponseEntity.ok(
                service.update(id, user));
    }

    // DELETE User
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(
            @PathVariable Long id) {

        service.delete(id);

        return ResponseEntity.noContent().build();
    }
}
```

# 9. CONTENT NEGOTIATION

Spring Boot supports:
- JSON
- XML
- Plain text

based on `Accept` header.

---

| Accept Header | Response Type |
|---|---|
| `application/json` | JSON |
| `application/xml` | XML |
| `text/plain` | Plain Text |
| `*/*` | Any |

---

## Example

```http
GET /api/users
Accept: application/json
```

---

## Default Behavior

Spring Boot APIs return JSON by default.
