# 1. VALIDATION IN SPRING BOOT

Validation ensures incoming data is:

- Correct
- Complete
- In expected format

---

## How Validation Works

1. Client sends request with data
2. Spring validates data using Bean Validation
3. If valid → proceed to business logic
4. If invalid → return error response

---

## Maven Dependency

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

---

## JSR-380 / Bean Validation

- Java standard for validation
- Implemented by Hibernate Validator
- Provides annotations for validating beans

---

# 2. COMMON VALIDATION ANNOTATIONS

| Annotation | Purpose | Example | Default Message |
|---|---|---|---|
| `@NotNull` | Value must not be null | `@NotNull private String name;` | must not be null |
| `@NotBlank` | String must not be blank | `@NotBlank private String name;` | must not be blank |
| `@NotEmpty` | Collection/String must not be empty | `@NotEmpty private List<String> list;` | must not be empty |
| `@Size(min,max)` | Size range validation | `@Size(min = 2, max = 10)` | size must be between 2 and 10 |
| `@Min` | Minimum numeric value | `@Min(1) private int age;` | must be greater than or equal to 1 |
| `@Max` | Maximum numeric value | `@Max(100) private int age;` | must be less than or equal to 100 |
| `@Email` | Valid email format | `@Email private String email;` | must be a well-formed email |
| `@Pattern` | Regex validation | `@Pattern(regexp="[A-Z]{3}")` | regex mismatch |

# 3. USING VALIDATION IN CONTROLLER

---

## Validate Request Body

```java
@PostMapping("/users")
public ResponseEntity<?> createUser(
        @Valid @RequestBody UserRequest request,
        BindingResult result) {

    if (result.hasErrors()) {
        // handle validation errors
    }

    // proceed...
}
```

---

## Validate PathVariable / RequestParam

```java
@GetMapping("/users/{id}")
public ResponseEntity<?> getUser(
        @PathVariable @Min(1) Long id) {

    // ...
}
```

---

```java
@GetMapping("/search")
public ResponseEntity<?> search(
        @RequestParam @NotBlank String name) {

    // ...
}
```

---

## Important Tip

- Use `@Valid` with `@RequestBody`
- Use `@Validated` for method-level validation

# 5. EXCEPTION HANDLING IN SPRING BOOT

---

## Why Exception Handling?

- Prevents stack trace exposure
- Provides meaningful error messages
- Centralizes error handling
- Improves API reliability

---

## Types of Exceptions

### Built-in Exceptions

Examples:
- `NullPointerException`
- `IllegalArgumentException`

---

### Custom Business Exceptions

Example:
- `UserNotFoundException`

---

### Validation Exceptions

Example:
- `MethodArgumentNotValidException`

---

## Approaches

| Approach | Scope |
|---|---|
| `@ExceptionHandler` | Specific controller |
| `@ControllerAdvice` | Global exception handling |

> `@ControllerAdvice` is the recommended approach.

---

# 6. @EXCEPTIONHANDLER (CONTROLLER LEVEL)

Handles exceptions only for a specific controller.

---

## Example

```java
@RestController
public class UserController {

    @GetMapping("/users/{id}")
    public User getUser(@PathVariable Long id) {

        throw new UserNotFoundException("User not found");
    }

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(
            UserNotFoundException ex) {

        ErrorResponse error =
                new ErrorResponse(
                        "USER_NOT_FOUND",
                        ex.getMessage());

        return new ResponseEntity<>(
                error,
                HttpStatus.NOT_FOUND);
    }
}
```

---

## Use Case

Use when exception handling is needed only for one controller.

---

# 7. @CONTROLLERADVICE (GLOBAL HANDLING)

Handles exceptions globally across all controllers.

---

## Example

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<?> handleUserNotFound(
            UserNotFoundException ex) {

        return ResponseEntity
                .status(HttpStatus.NOT_FOUND)
                .body(new ErrorResponse(
                        "USER_NOT_FOUND",
                        ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<?> handleValidationErrors(
            MethodArgumentNotValidException ex) {

        List<FieldErrorResponse> errors =
                ex.getBindingResult()
                  .getFieldErrors()
                  .stream()
                  .map(error ->
                          new FieldErrorResponse(
                                  error.getField(),
                                  error.getDefaultMessage()))
                  .toList();

        return ResponseEntity
                .badRequest()
                .body(new ErrorResponse(
                        "VALIDATION_FAILED",
                        "Validation failed",
                        errors));
    }
}
```

---

## Benefits

- Centralized exception handling
- Reusable code
- Cleaner controllers
- Consistent error responses

# 8. CUSTOM ERROR RESPONSE STRUCTURE

## ErrorResponse Class

```java
public class ErrorResponse {

    private String code;
    private String message;
    private List<FieldErrorResponse> errors;

    public ErrorResponse(
            String code,
            String message,
            List<FieldErrorResponse> errors) {

        this.code = code;
        this.message = message;
        this.errors = errors;
        this.timestamp = LocalDateTime.now();
```

# 10. BEST PRACTICES

- Use `@ControllerAdvice` for global handling
- Return meaningful error codes/messages
- Never expose stack traces in production
- Log exception details on server side
- Maintain consistent error response format
- Use proper HTTP status codes
- Create custom exceptions for business logic

| Status Code | Meaning | Use Case |
|---|---|---|

| 400 Bad Request | Invalid input | Validation failure |
| 401 Unauthorized | Authentication required | Missing/invalid login |
| 403 Forbidden | Access denied | No permission |
| 404 Not Found | Resource missing | Invalid ID |
| 500 Internal Server Error | Unexpected server error | Unhandled exception |
