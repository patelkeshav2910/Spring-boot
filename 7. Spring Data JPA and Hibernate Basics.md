# QUICK RECAP

- JPA is a specification and Hibernate is its most popular implementation.
- Spring Data JPA simplifies data access layer development.
- It provides repository abstraction, query methods, and pagination.

---

# 1. JPA VS HIBERNATE

| JPA (Java Persistence API) | Hibernate (ORM Framework) |
|---|---|
| Specification (Interface) | Implementation (Provider) |
| Part of Java EE / Jakarta EE | Most popular JPA provider |
| Defines ORM standards | Rich features & optimizations |
| Implemented by providers | Provides additional APIs |
| Examples: Hibernate, EclipseLink, OpenJPA | Handles SQL generation, caching, lazy loading |

---

## In Short

> JPA is what you write, Hibernate is what executes.

---

# 2. ORM OVERVIEW

```text
Java Objects
      ↓
ORM (Hibernate)
      ↓
Database Tables
```

---

## Benefits of ORM

- Eliminates JDBC boilerplate code
- Object-oriented interaction with database
- Database portability
- Automatic SQL generation
- Supports:
  - Caching
  - Lazy loading
  - Relationship management

---

# 3. ENTITY LIFECYCLE STATES

```text
New / Transient
       ↓ persist()
Managed / Persistent
       ↓ detach()
Detached
       ↓ remove()
Removed
```

---

## Lifecycle States

| State | Description |
|---|---|
| New / Transient | Object not associated with persistence context |
| Managed / Persistent | Object managed by Hibernate; changes tracked |
| Detached | Object exists but not connected to persistence context |
| Removed | Entity marked for deletion |

---

## Additional Operation

```text
merge()
```

Used to reattach detached entity.

# 4. IMPORTANT JPA ANNOTATIONS

| Annotation | Purpose | Example |
|---|---|---|
| `@Entity` | Marks class as JPA entity | `@Entity` |
| `@Table` | Specifies table details | `@Table(name = "users")` |
| `@Id` | Primary key | `@Id` |
| `@GeneratedValue` | PK generation strategy | `@GeneratedValue(strategy = GenerationType.IDENTITY)` |
| `@Column` | Maps field to column | `@Column(name = "user_name")` |
| `@Transient` | Field not persisted | `@Transient` |
| `@Temporal` | Date/time mapping | `@Temporal(TemporalType.DATE)` |
| `@Enumerated` | Enum mapping | `@Enumerated(EnumType.STRING)` |
| `@Lob` | Large object mapping | `@Lob` |
| `@Version` | Optimistic locking | `@Version` |

---

# 5. ENTITY EXAMPLE

```java
@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String email;

    @Column(name = "user_name")
    private String name;

    @Enumerated(EnumType.STRING)
    private Role role;

    @Temporal(TemporalType.TIMESTAMP)
    private Date createdAt;

    @Version
    private Integer version;
}
```

# 6. SPRING DATA JPA REPOSITORIES

Spring Data JPA provides repository abstraction for CRUD operations.

---

# Repository Hierarchy

```text
Repository
    ↓
CrudRepository
    ↓
PagingAndSortingRepository
    ↓
JpaRepository
```

---

# JpaRepository Features

- Ready-to-use CRUD methods
- Pagination & sorting
- Query methods
- Batch operations
- Custom queries

---

## Example Repository

```java
public interface UserRepository
        extends JpaRepository<User, Long> {

    List<User> findByEmail(String email);

    List<User> findByRole(Role role);
}
```

# 7. CRUD OPERATIONS (JPAREPOSITORY)

| Method | Description | Example |
|---|---|---|
| `save(entity)` | Save single entity | `repository.save(user)` |
| `saveAll(list)` | Save multiple entities | `repository.saveAll(users)` |
| `findById(id)` | Find entity by ID | `repository.findById(1L)` |
| `existsById(id)` | Check existence | `repository.existsById(1L)` |
| `findAll()` | Fetch all entities | `repository.findAll()` |
| `findAll(sort)` | Fetch sorted data | `repository.findAll(Sort.by("name"))` |
| `findAll(pageable)` | Pagination | `repository.findAll(PageRequest.of(0, 10))` |
| `count()` | Count records | `repository.count()` |
| `deleteById(id)` | Delete by ID | `repository.deleteById(1L)` |
| `delete(entity)` | Delete entity | `repository.delete(user)` |
| `deleteAll()` | Delete all records | `repository.deleteAll()` |

# 8. QUERY METHODS (DERIVED QUERIES)

Spring Data JPA automatically creates queries based on method names.

---

# Common Keywords

| Keyword | Example Method | Generated Query |
|---|---|---|
| `findBy` | `findByName(String name)` | `WHERE name = ?` |
| `findByAnd` | `findByNameAndEmail(...)` | `WHERE name = ? AND email = ?` |
| `Or` | `findByNameOrEmail(...)` | `WHERE name = ? OR email = ?` |
| `Between` | `findByAgeBetween(...)` | `WHERE age BETWEEN ? AND ?` |
| `LessThan` | `findByAgeLessThan(int age)` | `WHERE age < ?` |
| `Like` | `findByNameLike(String name)` | `WHERE name LIKE ?` |
| `OrderBy` | `findByNameOrderByAgeDesc()` | `ORDER BY age DESC` |
| `In` | `findByRoleIn(List<Role> roles)` | `WHERE role IN (...)` |

# 9. JPQL BASICS

JPQL (Java Persistence Query Language) is object-oriented query language for JPA.

---

## Example

```java
@Query("SELECT u FROM User u WHERE u.role = :role")
List<User> findByRole(
        @Param("role") Role role);
```

---

## Important Note

- JPQL works on entities, not tables.
- Used for complex queries.

---

# 10. NAMED QUERIES

Reusable queries defined using `@NamedQuery`.

---

## Example

```java
@Entity
@NamedQuery(
    name = "User.findByEmail",
    query = "SELECT u FROM User u WHERE u.email = :email"
)
public class User {

    @Id
    private Long id;

    private String email;
}
```

---

## Usage

```java
entityManager
    .createNamedQuery("User.findByEmail", User.class)
    .setParameter("email", "abc@gmail.com")
    .getResultList();
```

# 11. FIRST LEVEL CACHE (PERSISTENCE CONTEXT)

Hibernate maintains first-level cache at Session/PersistenceContext level.

---

```text
Database
    ↓
Persistence Context (1st Level Cache)
    ↓
Entity Object
```

---

## Benefits

- Same entity fetched multiple times → returned from cache
- Improves performance
- Scope:
  - Transaction/session level

---

## Cache Cleanup

Automatically cleared after:
- Transaction commit
- Rollback

---

# 12. HOW save() WORKS INTERNALLY

```text
Entity Object
      ↓
Persistence Context
      ↓
Dirty Checking
      ↓
SQL Generation
      ↓
Database
```

---

## Internal Flow

1. Entity added to persistence context
2. Hibernate tracks changes (Dirty Checking)
3. On flush/commit:
   - SQL generated automatically
4. Query executed against DB

# PRO TIPS

- Prefer repositories over direct `EntityManager`.
- Enable SQL logging during development.
- Understand Lazy Loading and N+1 problem deeply.
- Use DTO projections for performance optimization.
- Keep entities clean and focused on persistence.

# BEST PRACTICES

- Use meaningful entity and field names.
- Prefer `FetchType.LAZY` for collections.
- Use `@Transactional` at service layer.
- Create custom queries only when required.
- Avoid unnecessary eager loading.
