# QUICK RECAP

- JPA relationships define how entities are associated.
- Proper mapping improves data integrity and reduces redundancy.
- Fetch strategies and cascading affect performance and behavior.
- Understand:
  - Owning side
  - `mappedBy`

# 1. TYPES OF RELATIONSHIPS

| Relationship | Description |
|---|---|
| OneToOne | One entity associated with one other entity |
| OneToMany | One entity associated with many entities |
| ManyToOne | Many entities associated with one entity |
| ManyToMany | Many entities associated with many entities |

---

## Relationship Examples

```text
OneToOne
User ↔ Profile
```

```text
OneToMany
User → Posts
```

```text
ManyToOne
Post → User
```

```text
ManyToMany
Student ↔ Course
```

---

# 2. OWNING SIDE VS INVERSE SIDE

---

## Owning Side

- Contains foreign key
- Changes are persisted to DB
- Usually contains `@JoinColumn`

---

## Inverse Side (`mappedBy`)

- Opposite side of relationship
- Does not own foreign key
- Uses `mappedBy`

---

## Important Rule

> Always update the owning side of the relationship.

---

# 3. CASCADE TYPES

| Cascade Type | Description | Use Case |
|---|---|---|
| `CascadeType.PERSIST` | Propagates persist operation | Save parent → save child |
| `CascadeType.MERGE` | Propagates merge operation | Update parent → update child |
| `CascadeType.REMOVE` | Propagates delete operation | Delete parent → delete child |
| `CascadeType.REFRESH` | Refreshes child entities | Reload latest DB state |
| `CascadeType.DETACH` | Detaches child entities | Remove from persistence context |
| `CascadeType.ALL` | Includes all cascade operations | Use carefully |

---

## Tip

Use `CascadeType.ALL` carefully to avoid accidental deletes.

# 4. RELATIONSHIP MAPPINGS WITH EXAMPLES

---

# 4.1 OneToOne

```text
User ↔ Profile
```

- One user has one profile
- Uses `@OneToOne`

---

## User.java (Owning Side)

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "profile_id")
    private Profile profile;
}
```

---

## Profile.java (Inverse Side)

```java
@Entity
public class Profile {

    @Id
    @GeneratedValue
    private Long id;

    private String bio;

    @OneToOne(mappedBy = "profile")
    private User user;
}
```

---

## Key Points

- Owning side uses `@JoinColumn`
- Inverse side uses `mappedBy`

---

# 4.2 OneToMany

```text
User → Posts
```

- One user can have many posts
- Parent side uses `@OneToMany`

---

## User.java (Inverse Side)

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToMany(
            mappedBy = "user",
            cascade = CascadeType.ALL,
            orphanRemoval = true
    )
    private List<Post> posts = new ArrayList<>();
}
```

---

## Post.java (Owning Side)

```java
@Entity
public class Post {

    @Id
    @GeneratedValue
    private Long id;

    private String title;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private User user;
}
```

---

## Key Points

- `@ManyToOne` is usually owning side
- `mappedBy` defines inverse side
- `orphanRemoval = true` deletes child when removed from collection

---

# 4.3 ManyToOne

```text
Post → User
```

- Many posts belong to one user
- Child entity usually owns relationship

---

## Example

```java
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "user_id", nullable = false)
private User user;
```

---

## Key Points

- Foreign key stored in child table
- `@ManyToOne` defaults to `EAGER`
- Prefer `LAZY` for performance

---

# 4.4 ManyToMany

```text
Student ↔ Course
```

- Many students can enroll in many courses
- Uses join table

---

## Student.java

```java
@Entity
public class Student {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @ManyToMany(cascade = CascadeType.ALL)
    @JoinTable(
            name = "student_course",
            joinColumns = @JoinColumn(name = "student_id"),
            inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses;
}
```

---

## Course.java

```java
@Entity
public class Course {

    @Id
    @GeneratedValue
    private Long id;

    private String title;

    @ManyToMany(mappedBy = "courses")
    private List<Student> students;
}
```

---

## Key Points

- JPA creates join table automatically
- Use `mappedBy` on one side
- Avoid unnecessary bidirectional mappings

---

# 5. FETCH TYPES

| Fetch Type | Description | When to Use |
|---|---|---|
| `LAZY` | Loads related data only when accessed | Best for performance |
| `EAGER` | Loads related data immediately | Use only when always needed |

---

## Default Fetch Types

| Relationship | Default Fetch |
|---|---|
| `@OneToMany` | LAZY |
| `@ManyToMany` | LAZY |
| `@ManyToOne` | EAGER |
| `@OneToOne` | EAGER |

---

## Warning

```text
Lazy loading outside transaction
→ LazyInitializationException
```

---

# 6. FETCH JOIN (SOLVING N+1 PROBLEM)

Use JPQL `JOIN FETCH` to load associations in single query.

---

## Example

```java
@Query("""
       SELECT u
       FROM User u
       JOIN FETCH u.posts
       WHERE u.id = :id
       """)
User findUserWithPosts(@Param("id") Long id);
```

---

## Without Fetch Join

```text
1 query → User
N queries → Posts
```

Result:
```text
N + 1 Problem
```

---

## With Fetch Join

```text
1 query → User + Posts
```

---

# 7. N+1 SELECT PROBLEM

Occurs when:
- Parent entities fetched first
- Child entities loaded separately in loop

---

## Example Scenario

```text
Fetch 10 users
↓
Fetch posts for each user separately
↓
1 + 10 = 11 queries
```

---

## Solutions

- Use `JOIN FETCH`
- Use `@EntityGraph`
- Use batch fetching
- Use DTO projections

# 8. MAPPING ANNOTATIONS QUICK REFERENCE

| Annotation | Used In | Purpose |
|---|---|---|
| `@OneToOne` | Entity | One-to-one mapping |
| `@OneToMany` | Entity | One-to-many mapping |
| `@ManyToOne` | Entity | Many-to-one mapping |
| `@ManyToMany` | Entity | Many-to-many mapping |
| `@JoinColumn` | Owning Side | Defines foreign key |
| `@JoinTable` | ManyToMany | Defines join table |
| `mappedBy` | Inverse Side | Defines non-owning side |
| `fetch` | Relationship | Fetch strategy |
| `cascade` | Relationship | Cascade operations |
| `orphanRemoval` | OneToMany | Removes detached children |

---

# 9. CASCADE + ORPHAN REMOVAL EXAMPLE

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    @OneToMany(
            mappedBy = "user",
            cascade = CascadeType.ALL,
            orphanRemoval = true
    )
    private List<Post> posts = new ArrayList<>();
}
```

---

## Behavior

- Save user → saves posts
- Delete user → deletes posts
- Remove post from list → post deleted from DB

# PRO TIPS

- Prefer `LAZY` loading for better performance
- Always define both sides carefully
- Use appropriate cascade types
- Avoid `CascadeType.ALL` with `@ManyToMany`
- Use DTOs for read-heavy operations
